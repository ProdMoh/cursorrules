## üöÄ How It Works

### Step 1: Create a PRD on ProdMoh

1. Go to [prodmoh.com](https://prodmoh.com) and sign in
2. Click **"New PRD"** (Product Requirements Document)
3. Describe your feature or product idea
4. ProdMoh generates a complete PRD with:
   - Product vision & strategy
   - Target audience & personas
   - Feature requirements
   - Success metrics

### Step 2: Generate User Stories

1. Open your PRD
2. Click **"Generate User Stories"**
3. ProdMoh creates detailed user stories with:
   - Acceptance criteria
   - Technical considerations
   - Edge cases

### Step 3: Configure AI Governance

1. From your PRD or User Story, click **"Governance Config"**
2. Select your **industry preset**:
   - üè• **Healthcare** ‚Üí HIPAA + OWASP security, audit logging
   - üí∞ **Fintech** ‚Üí SOC2 + PCI-DSS, structured logging
   - üíª **General SaaS** ‚Üí OWASP Top 10, clean architecture
3. Customize additional options:
   - Design Pattern (MVC, Clean, Hexagonal)
   - Security Framework
   - Logging Standard
   - Testing Strategy

### Step 4: Push Directly to GitHub

1. Click **"Export Governance Config"** ‚Üí **"Push to GitHub"**
2. **First time only:** Connect your GitHub account (OAuth)
3. Enter your repository (e.g., `your-org/backend-service`)
4. Select the file type:
   - `.cursorrules` for Cursor
   - `.windsurfrules` for Windsurf
   - `copilot-instructions.md` for GitHub Copilot
   - `.moh` for universal AI assistants
5. Click **"Push to Repo"** ‚Üí ‚úÖ Done!
